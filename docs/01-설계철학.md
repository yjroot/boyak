# 보약 프로그래밍 언어 설계 철학

## 1. 언어 이름의 의미

**"보약"**은 한국 전통 의학에서 몸을 보하고 건강하게 하는 약을 의미합니다. 이 이름을 선택한 이유:

1. **친근함**: 한국인에게 익숙하고 친근한 단어
2. **긍정적 이미지**: "도움이 되는", "좋은 것"이라는 연상
3. **은유적 의미**: 프로그래밍을 더 쉽고 건강하게 만들어준다는 의미
4. **발음의 용이성**: 두 음절로 짧고 발음하기 쉬움

## 2. 핵심 설계 원칙

### 2.1 한글 우선 원칙 (Korean-First Principle)

**근거**: 한글은 세계에서 가장 과학적으로 설계된 문자 체계 중 하나입니다. 한글의 논리적 구조를 프로그래밍 언어에 반영하면 한국어 화자에게 직관적인 코드를 작성할 수 있습니다.

- 모든 키워드는 한글로 작성
- 영어 키워드의 직역이 아닌, 한국어 사고방식에 맞는 표현 사용
- 조사(은/는, 이/가, 을/를 등)를 문법의 일부로 활용

### 2.2 SOV 어순 존중 (Subject-Object-Verb Order)

**근거**: 한국어는 SOV(주어-목적어-동사) 어순을 따릅니다. 대부분의 프로그래밍 언어는 SVO 또는 VSO 어순(영어 기반)을 사용하는데, 이는 한국어 화자에게 부자연스럽습니다.

```
영어: print("Hello")     → 동사(print) + 목적어("Hello")
보약: "안녕"을 출력하라   → 목적어("안녕") + 조사(을) + 동사(출력하라)
```

### 2.3 경어법 활용 (Honorific System)

**근거**: 한국어의 경어법은 문장의 종류(명령, 청유, 서술 등)를 구분하는 데 자연스럽게 활용될 수 있습니다.

| 종결어미 | 의미 | 프로그래밍 맥락 |
|---------|------|----------------|
| ~하라 | 명령 | 함수 호출, 실행문 |
| ~하자 | 청유 | 반복문, 공동 작업 |
| ~이다/다 | 서술 | 변수 선언, 정의 |
| ~인가? | 의문 | 조건 검사, 불리언 표현 |

### 2.4 들여쓰기 기반 블록 (Python 스타일)

**근거**:
- Python의 들여쓰기 기반 문법은 초보자에게 친숙함
- 중괄호 `{}`보다 시각적으로 깔끔함
- 한글 텍스트와 자연스럽게 어울림
- 코드의 논리적 구조가 시각적으로 드러남

### 2.5 최소 기호 원칙 (Minimal Symbol Principle)

**근거**: 한글 키보드에서 특수 기호 입력은 번거롭습니다. 가능한 한 자연어에 가까운 표현을 사용합니다.

```
기존 언어: if (x > 5 && y < 10)
보약:      만약 x가 5보다 크고 y가 10보다 작으면
```

## 3. 대상 사용자

1. **프로그래밍 입문자**: 영어 장벽 없이 프로그래밍 개념 학습
2. **한국어 교육**: 프로그래밍을 통한 논리적 사고 교육
3. **비전공자**: 업무 자동화, 간단한 스크립트 작성
4. **어린이**: 컴퓨팅 사고력 교육

## 4. 영향을 받은 언어들

| 언어 | 영향 받은 요소 |
|------|---------------|
| Python | 들여쓰기 기반 블록, 간결한 문법, 동적 타입 |
| Ruby | 자연어에 가까운 문법, 블록 표현 |
| AppleScript | 자연어 스타일 프로그래밍 |
| 한글 프로그래밍 언어들 (씨앗, 아희) | 한글 키워드 사용의 선례 |

## 5. 설계 결정 시 우선순위

1. **가독성** > 작성 편의성 > 실행 효율성
2. **직관성** > 기능의 풍부함
3. **일관성** > 편의를 위한 예외
4. **안전성** > 유연성

## 6. 언어의 범위

### 포함하는 것
- 기본 데이터 타입 (숫자, 문자열, 불리언, 목록, 사전)
- 변수와 상수
- 제어 구조 (조건문, 반복문)
- 함수 정의와 호출
- 기본 입출력
- 모듈 시스템 (간단한 형태)

### 포함하지 않는 것 (1.0 버전)
- 클래스와 객체지향 프로그래밍 (2.0에서 추가 예정)
- 비동기 프로그래밍
- 저수준 메모리 조작
- 복잡한 타입 시스템

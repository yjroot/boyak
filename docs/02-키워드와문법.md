# 보약 키워드 및 기본 문법

## 1. 키워드 설계 원칙

### 1.1 한국어 문법 구조 활용

한국어는 교착어(agglutinative language)로, 어근에 조사와 어미가 붙어 의미를 완성합니다. 보약은 이 특성을 활용합니다.

```
어근 + 조사/어미 = 완전한 의미

예: 출력 + 하라 = 출력하라 (명령: print)
    반복 + 하자 = 반복하자 (청유: loop)
```

## 2. 키워드 목록

### 2.1 변수와 상수

| 키워드 | 역할 | 근거 |
|--------|------|------|
| `은/는` | 변수 정의 (주제 표시) | 한국어에서 '은/는'은 주제를 나타내며, "X는 Y이다"는 정의 문장 |
| `이다` | 할당 완료 표시 | 서술형 종결어미로 선언문 완성 |
| `항상` | 상수 선언 | "항상 같은 값"이라는 직관적 의미 |

**문법 예시:**
```보약
나이는 25이다
이름은 "홍길동"이다
항상 원주율은 3.14159이다
```

**설계 근거:**
- "X는 Y이다"는 한국어의 가장 기본적인 정의 문장 구조
- 조사 '은/는'을 사용하여 할당 연산자(=)를 대체
- 읽으면 자연스러운 한국어 문장이 됨

### 2.2 조건문

| 키워드 | 역할 | 대응 개념 |
|--------|------|----------|
| `만약` | 조건 시작 | if |
| `이면` / `면` | 조건 종료 | then (암시적) |
| `아니면` | else if | elif |
| `그외에` | 기본 분기 | else |

**문법 예시:**
```보약
만약 나이가 18보다 크거나같으면
    "성인입니다"를 출력하라
아니면 나이가 13보다 크거나같으면
    "청소년입니다"를 출력하라
그외에
    "어린이입니다"를 출력하라
```

**설계 근거:**
- "만약 ~이면"은 한국어에서 조건을 표현하는 자연스러운 구조
- 조건절 끝에 '면/이면'을 붙여 조건의 끝을 명확히 표시
- "그외에"는 "그 외의 경우에"의 줄임말로 직관적

### 2.3 반복문

| 키워드 | 역할 | 대응 개념 |
|--------|------|----------|
| `반복하자` | while 루프 시작 | while |
| `동안` | 조건 반복 | while condition |
| `각각의` | for-each 시작 | for ... in |
| `에서` | 범위/컬렉션 지정 | in |
| `부터 ~ 까지` | 범위 지정 | range |
| `그만` | 루프 탈출 | break |
| `다음으로` | 다음 반복으로 | continue |

**문법 예시:**
```보약
# while 스타일
횟수는 0이다
횟수가 10보다 작은동안 반복하자
    횟수를 출력하라
    횟수는 횟수 + 1이다

# for-each 스타일
과일들은 ["사과", "배", "포도"]이다
각각의 과일에서 과일들을 반복하자
    과일을 출력하라

# 범위 반복
1부터 10까지 반복하자 숫자로
    숫자를 출력하라
```

**설계 근거:**
- "~하자"는 청유형으로, "함께 이 작업을 반복하자"는 뉘앙스
- "~동안"은 시간/조건의 지속을 나타내는 자연스러운 표현
- "각각의"는 컬렉션의 개별 요소를 지칭하는 명확한 표현
- "부터 ~ 까지"는 범위를 나타내는 한국어의 기본 구조

### 2.4 함수

| 키워드 | 역할 | 대응 개념 |
|--------|------|----------|
| `정의하자` | 함수 정의 시작 | def |
| `을/를 받아` | 매개변수 선언 | parameters |
| `돌려주라` | 값 반환 | return |
| `아무것도 없이` | 매개변수 없음 | no parameters |

**문법 예시:**
```보약
# 매개변수가 있는 함수
인사를 정의하자 이름을 받아
    "안녕하세요, " + 이름 + "님!"을 돌려주라

# 매개변수가 없는 함수
현재시각을 정의하자 아무것도없이
    지금을 돌려주라

# 여러 매개변수
더하기를 정의하자 첫째와 둘째를 받아
    첫째 + 둘째를 돌려주라

# 함수 호출
결과는 인사("홍길동")이다
합계는 더하기(3, 5)이다
```

**설계 근거:**
- "정의하자"는 "이것을 정의하자"라는 청유형으로 함수 선언을 나타냄
- "~을/를 받아"는 함수가 입력을 받는다는 것을 명확히 표현
- "돌려주라"는 "결과를 돌려주라"는 명령으로 return의 의미를 직관적으로 전달
- 함수 호출은 Python과 유사하게 괄호 사용 (보편성과 타 언어 학습 연계)

### 2.5 입출력

| 키워드 | 역할 | 대응 개념 |
|--------|------|----------|
| `출력하라` | 화면 출력 | print |
| `입력받아라` | 사용자 입력 | input |
| `줄바꿈없이` | 줄바꿈 없는 출력 | end="" |

**문법 예시:**
```보약
"안녕하세요"를 출력하라
이름은 "이름을 입력하세요: "로 입력받아라
"환영합니다, "를 줄바꿈없이 출력하라
이름을 출력하라
```

**설계 근거:**
- "~하라"는 명령형 종결어미로 실행 명령을 나타냄
- "출력하라/입력받아라"는 동작을 명확히 설명하는 동사
- 목적어 + 조사(을/를) + 동사 구조로 한국어 어순 유지

### 2.6 논리 연산자

| 키워드 | 역할 | 대응 개념 |
|--------|------|----------|
| `그리고` / `이고` | 논리 AND | and |
| `또는` / `이거나` | 논리 OR | or |
| `아닌` / `이 아닌` | 논리 NOT | not |
| `참` | True | True |
| `거짓` | False | False |

**문법 예시:**
```보약
만약 나이가 18보다 크고 학생이면
    "성인 학생입니다"를 출력하라

만약 비가오거나 눈이오면
    "우산을 가져가세요"를 출력하라

만약 회원이 아니면
    "가입이 필요합니다"를 출력하라
```

**설계 근거:**
- "그리고/또는"은 한국어에서 접속을 나타내는 기본 표현
- "이고/이거나"는 술어에 붙는 연결어미로 더 자연스러운 표현 가능
- "아닌"은 부정을 나타내는 가장 간결한 표현

### 2.7 비교 연산자

| 표현 | 역할 | 대응 기호 |
|------|------|----------|
| `같다` / `같으면` | 동등 비교 | == |
| `다르다` / `다르면` | 비동등 비교 | != |
| `보다 크다` / `보다 크면` | 크기 비교 | > |
| `보다 작다` / `보다 작으면` | 크기 비교 | < |
| `보다 크거나같다` | 이상 비교 | >= |
| `보다 작거나같다` | 이하 비교 | <= |

**문법 예시:**
```보약
만약 점수가 90보다 크거나같으면
    "A학점"을 출력하라

만약 이름이 "관리자"와 같으면
    "관리자 모드"를 출력하라

만약 상태가 "완료"와 다르면
    "진행 중"을 출력하라
```

**설계 근거:**
- 한국어의 비교 표현을 그대로 사용
- "~보다 크다/작다"는 수학적 비교의 자연어 표현
- 기호 대신 단어를 사용하여 가독성 향상

## 3. 조사 처리 규칙

### 3.1 주격 조사 (이/가)
- 주어를 나타낼 때 사용
- 조건문에서 주어 뒤에 사용: "나이**가** 18보다 크면"

### 3.2 목적격 조사 (을/를)
- 동작의 대상을 나타낼 때 사용
- 함수 호출, 출력문에서 사용: "결과**를** 출력하라"

### 3.3 보조사 (은/는)
- 주제를 나타내거나 변수 정의에 사용
- "나이**는** 25이다"

### 3.4 조사 생략 규칙
```보약
# 다음은 모두 동일하게 처리됨
"안녕"을 출력하라
"안녕" 출력하라      # 조사 생략 가능
출력하라 "안녕"      # 어순 변경 가능 (대안 문법)
```

**설계 근거:**
- 한국어에서 조사는 문맥상 생략 가능한 경우가 많음
- 초보자의 진입 장벽을 낮추기 위해 조사 생략 허용
- 단, 명확성을 위해 조사 사용 권장

## 4. 예약어 전체 목록

```
# 값
참, 거짓, 없음

# 변수/상수
은, 는, 이다, 항상

# 조건
만약, 이면, 면, 아니면, 그외에

# 반복
반복하자, 동안, 각각의, 에서, 부터, 까지, 그만, 다음으로

# 함수
정의하자, 받아, 돌려주라, 아무것도없이

# 입출력
출력하라, 입력받아라, 줄바꿈없이

# 논리
그리고, 이고, 또는, 이거나, 아닌

# 비교
같다, 다르다, 보다, 크다, 작다, 크거나같다, 작거나같다

# 기타
포함하라
```

## 5. 주석

```보약
# 한 줄 주석은 샵으로 시작

##
여러 줄 주석은
샵 두 개로 감싼다
##
```

**설계 근거:**
- `#`은 Python과 동일하여 학습 연계성 유지
- 여러 줄 주석에 `## ~ ##` 사용으로 시작과 끝 명확히 구분

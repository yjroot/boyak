# 보약 데이터 타입 시스템

## 1. 타입 시스템 철학

### 1.1 동적 타입 (Dynamic Typing)

**선택 근거:**
- Python처럼 간단하고 유연한 사용성 제공
- 초보자가 타입 선언 없이 바로 코딩 시작 가능
- 빠른 프로토타이핑에 적합
- 한국어의 유연한 문법 구조와 조화

### 1.2 강타입 (Strong Typing)

**선택 근거:**
- 암시적 타입 변환으로 인한 버그 방지
- 명시적 변환 요구로 코드 의도 명확화
- 교육용 언어로서 올바른 습관 형성

```보약
# 오류: 숫자와 문자열은 자동 변환되지 않음
결과는 "나이: " + 25이다  # 오류!

# 올바른 방법: 명시적 변환
결과는 "나이: " + (25를 문자열로)이다
```

## 2. 기본 데이터 타입

### 2.1 수 (숫자)

| 한글명 | 설명 | Python 대응 |
|--------|------|-------------|
| `정수` | 정수형 | int |
| `실수` | 부동소수점 | float |

**리터럴 표기:**
```보약
나이는 25이다           # 정수
키는 175.5이다          # 실수
온도는 -10이다          # 음수
큰수는 1_000_000이다    # 천 단위 구분 (가독성)
```

**설계 근거:**
- "정수/실수"는 수학 용어로 한국인에게 익숙
- 밑줄로 천 단위 구분 허용 (Python 3.6+ 스타일)

### 2.2 문자열

| 한글명 | 설명 | Python 대응 |
|--------|------|-------------|
| `문자열` | 텍스트 데이터 | str |

**리터럴 표기:**
```보약
이름은 "홍길동"이다           # 큰따옴표
인사는 '안녕하세요'이다        # 작은따옴표
설명은 """여러 줄에 걸친
긴 문자열입니다"""이다         # 삼중 따옴표

# 문자열 보간 (템플릿 문자열)
메시지는 "{이름}님, 환영합니다!"이다

# 이스케이프 문자
줄바꿈은 "첫째 줄\n둘째 줄"이다
탭은 "이름\t나이"이다
```

**설계 근거:**
- Python과 동일한 따옴표 규칙으로 학습 연계
- 문자열 보간에 중괄호 사용 (f-string 스타일)
- 표준 이스케이프 문자 지원

### 2.3 불리언

| 한글명 | 설명 | Python 대응 |
|--------|------|-------------|
| `참` | 참 값 | True |
| `거짓` | 거짓 값 | False |

**문법 예시:**
```보약
활성화됨은 참이다
완료됨은 거짓이다

만약 활성화됨이면
    "시스템이 활성화되었습니다"를 출력하라
```

**설계 근거:**
- "참/거짓"은 한국어에서 논리값을 나타내는 가장 자연스러운 표현
- True/False의 직역(진실/허위)보다 직관적

### 2.4 없음 (Null)

| 한글명 | 설명 | Python 대응 |
|--------|------|-------------|
| `없음` | 값이 없음 | None |

**문법 예시:**
```보약
결과는 없음이다

만약 결과가 없음이면
    "결과가 없습니다"를 출력하라
```

**설계 근거:**
- "없음"은 None/null의 의미를 가장 직관적으로 표현
- "비어있음", "널"보다 간결하고 명확

## 3. 컬렉션 타입

### 3.1 목록 (List)

**특성:**
- 순서가 있음 (ordered)
- 수정 가능 (mutable)
- 중복 허용

**문법:**
```보약
# 목록 생성
과일들은 ["사과", "배", "포도"]이다
숫자들은 [1, 2, 3, 4, 5]이다
빈목록은 []이다

# 요소 접근 (0부터 시작)
첫번째는 과일들[0]이다        # "사과"
마지막은 과일들[-1]이다       # "포도" (음수 인덱스)

# 범위 접근 (슬라이싱)
일부는 숫자들[1:3]이다        # [2, 3]
앞부분은 숫자들[:3]이다       # [1, 2, 3]
뒷부분은 숫자들[2:]이다       # [3, 4, 5]

# 목록 조작
과일들에 "딸기"를 추가하라
과일들에서 "배"를 제거하라
길이는 과일들의 길이이다
```

**설계 근거:**
- 대괄호 `[]`는 보편적인 목록 표기법
- Python 스타일 슬라이싱으로 학습 연계
- 조작 메서드는 한글로 "추가하라/제거하라"

### 3.2 사전 (Dictionary)

**특성:**
- 키-값 쌍으로 구성
- 키로 빠른 접근
- 수정 가능

**문법:**
```보약
# 사전 생성
학생은 {
    "이름": "홍길동",
    "나이": 20,
    "학과": "컴퓨터공학"
}이다

빈사전은 {}이다

# 값 접근
이름은 학생["이름"]이다
나이는 학생의 나이이다        # 대안 문법

# 값 수정/추가
학생["학년"]은 3이다
학생의 나이는 21이다

# 키 존재 확인
만약 "이름"이 학생에 있으면
    학생["이름"]을 출력하라

# 사전 조작
학생에서 "학과"를 제거하라
모든키는 학생의 키들이다
모든값은 학생의 값들이다
```

**설계 근거:**
- 중괄호 `{}`는 보편적인 사전 표기법
- "X의 Y" 구문으로 점(.) 접근을 한글화
- "있으면" 구문으로 `in` 연산자 대체

### 3.3 집합 (Set)

**특성:**
- 순서 없음
- 중복 불허
- 수학적 집합 연산 지원

**문법:**
```보약
# 집합 생성
과일집합은 집합("사과", "배", "포도")이다
숫자집합은 집합(1, 2, 3, 4, 5)이다
빈집합은 집합()이다

# 집합 연산
합집합은 집합1 | 집합2이다        # 또는: 집합1과 집합2의 합집합
교집합은 집합1 & 집합2이다        # 또는: 집합1과 집합2의 교집합
차집합은 집합1 - 집합2이다        # 또는: 집합1에서 집합2를 뺀 것

# 집합 조작
과일집합에 "딸기"를 추가하라
과일집합에서 "배"를 제거하라
```

**설계 근거:**
- 빈 집합과 빈 사전 구분을 위해 `집합()` 함수 사용
- 수학적 집합 기호(|, &, -)를 그대로 사용하여 직관성 확보
- 한글 대안 문법도 제공

## 4. 타입 변환

### 4.1 변환 문법

```보약
# 기본 변환 구문: X를 Y(으)로
문자는 123을 문자열로이다           # "123"
숫자는 "456"을 정수로이다           # 456
실수값은 "3.14"를 실수로이다        # 3.14
목록값은 "안녕"을 목록으로이다      # ["안", "녕"]

# 대안 구문: Y(X)
문자는 문자열(123)이다
숫자는 정수("456")이다
```

**설계 근거:**
- "X를 Y로"는 한국어에서 변환을 나타내는 자연스러운 표현
- "물을 얼음으로 만들다"와 같은 구조
- 함수 스타일 변환도 지원하여 유연성 제공

### 4.2 타입 확인

```보약
# 타입 확인
만약 값이 정수이면
    "정수입니다"를 출력하라

만약 값이 문자열이면
    "문자열입니다"를 출력하라

# 타입 얻기
타입이름은 값의 타입이다
타입이름을 출력하라              # "정수", "문자열" 등
```

**설계 근거:**
- "X가 Y이면" 구문으로 타입 체크
- Python의 `isinstance()`와 `type()`에 대응

## 5. 연산자

### 5.1 산술 연산자

| 연산 | 기호 | 한글 대안 | 예시 |
|------|------|----------|------|
| 덧셈 | `+` | `더하기` | `3 + 5` 또는 `3 더하기 5` |
| 뺄셈 | `-` | `빼기` | `10 - 3` 또는 `10 빼기 3` |
| 곱셈 | `*` | `곱하기` | `4 * 5` 또는 `4 곱하기 5` |
| 나눗셈 | `/` | `나누기` | `10 / 2` 또는 `10 나누기 2` |
| 정수 나눗셈 | `//` | `몫` | `10 // 3` 또는 `10의 3으로 나눈 몫` |
| 나머지 | `%` | `나머지` | `10 % 3` 또는 `10의 3으로 나눈 나머지` |
| 거듭제곱 | `**` | `제곱` | `2 ** 3` 또는 `2의 3제곱` |

**설계 근거:**
- 수학 기호는 보편적이므로 기본 지원
- 한글 대안으로 코드 가독성 향상 옵션 제공
- 수학 교육과 연계 가능

### 5.2 복합 할당 연산자

```보약
# 기호 스타일
점수 += 10        # 점수는 점수 + 10이다 와 동일
점수 -= 5
점수 *= 2
점수 /= 2

# 한글 스타일
점수에 10을 더하라
점수에서 5를 빼라
점수를 2배로 하라
```

**설계 근거:**
- 기호 스타일은 기존 프로그래머에게 친숙
- 한글 스타일은 초보자에게 직관적
- 두 가지 모두 지원하여 선택권 제공

## 6. 타입 힌트 (선택적)

```보약
# 타입 힌트는 선택적이며, 문서화 목적
나이는 정수: 25이다
이름은 문자열: "홍길동"이다
점수들은 목록[정수]: [85, 90, 78]이다

# 함수에서의 타입 힌트
더하기를 정의하자 (첫째: 정수, 둘째: 정수) -> 정수로 받아
    첫째 + 둘째를 돌려주라
```

**설계 근거:**
- Python의 타입 힌트와 유사한 문법
- 강제가 아닌 선택적 사용
- 큰 프로젝트에서 코드 문서화에 유용
- IDE 지원 시 자동완성 등에 활용 가능
